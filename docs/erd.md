## Таблицы

### `users`

Хранит пользователей приложения.

| Колонка    | Тип       | PK/FK  | Примечание                                 |
| ---------- | --------- | ------ | ------------------------------------------ |
| id         | bigint    | PK     | Уникальный id пользователя                 |
| name       | text      |        | Имя пользователя                           |
| created_at | timestamp |        | Дата создания аккаунта                     |

### `relatives`

Хранит родственников, которых пользователь добавил в приложение.

| Колонка                | Тип       | PK/FK         | Примечание                                  |
| ---------------------- | --------- | ------------- | ------------------------------------------- |
| id                     | bigint    | PK            | Уникальный id родственника                  |
| user_id                | bigint    | FK → users.id | Какому пользователю принадлежит родственник |
| name                   | text      |               | Имя родственника                            |
| relation               | text      |               | Степень родства: мама, апа, ата, брат, тәте |
| reminder_interval_days | int       |               | Через сколько дней напоминать               |
| last_contact_at        | date      |               | Когда последний раз общались                |
| created_at             | timestamp |               | Дата добавления родственника                |

### `reminders`

Хранит напоминания о том, что пора связаться с родственником.

| Колонка     | Тип       | PK/FK             | Примечание                                  |
| ----------- | --------- | ----------------- | ------------------------------------------- |
| id          | bigint    | PK                | Уникальный id напоминания                   |
| relative_id | bigint    | FK → relatives.id | К какому родственнику относится напоминание |
| remind_at   | timestamp |                   | Когда нужно показать напоминание            |
| status      | text      |                   | Статус: pending, sent, done, skipped        |
| created_at  | timestamp |                   | Дата создания напоминания                   |

## Связи

* `users` 1 — N `relatives` через `relatives.user_id`
* `relatives` 1 — N `reminders` через `reminders.relative_id`

## Как фичи связаны с таблицами

### F1: Добавление родственника

Когда пользователь добавляет родственника, создаётся новая строка в таблице `relatives`.

### F2: Напоминание, с кем общаться

Система смотрит `last_contact_at` и `reminder_interval_days` в таблице `relatives`.
Если прошло достаточно дней, создаётся или используется запись в таблице `reminders`.

### F3: AI-текст для начала общения

В v1 AI-текст можно не хранить в отдельной таблице.
Приложение может просто генерировать или показывать готовый шаблон сообщения на экране.
